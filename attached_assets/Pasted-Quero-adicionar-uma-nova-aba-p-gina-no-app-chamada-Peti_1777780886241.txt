Quero adicionar uma nova aba/página no app chamada "🧾 Petições" entre "Movimentações" e "Financeiro" no menu lateral.

OBJETIVO
A página mostra quantas petições cada responsável fez (HOJE / ONTEM / ESTA SEMANA / ESTE MÊS). É a métrica de produtividade da equipe jurídica.

ACESSO
- Admin: vê TUDO (todos os responsáveis, totais consolidados, drill-down completo)
- Team (Thiago, Tammyres, etc): vê APENAS as próprias petições (filtrar pelo usuário logado). Reusa o mesmo padrão de admin-only que já existe no projeto.

BACKEND — ROTA E DADOS

1. Cria a rota GET /api/petitions/by-person
   - Query params: period (today | yesterday | this_week | this_month), responsible (opcional, nome)
   - Default: period=today
   
2. Fonte de dados: AdvBox API.
   - Use o cliente AdvBox que JÁ EXISTE no código (não crie novo). Procure no projeto por funções tipo getMovimentacoes / fetchAndamentos / similar.
   - Filtra movimentações por TIPO = "Petição" (ou equivalente — verifica como o AdvBox tipa isso, pode ser "Andamento" com subtipo "Peça Processual").
   - Agrupa por responsável e conta.
   
3. Resposta JSON (exemplo):
   {
     "period": "today",
     "total": 12,
     "by_person": [
       { "name": "Maria Alice", "count": 4, "types": { "Inicial": 2, "Recurso": 1, "Cumprimento": 1 } },
       { "name": "Letícia Stephany", "count": 6, "types": { "Inicial": 3, "Manifestação": 3 } },
       { "name": "Claudiana", "count": 2, "types": { "Recurso": 2 } }
     ]
   }
